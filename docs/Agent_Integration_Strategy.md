# สรุปแนวทางการพัฒนา: การเชื่อมต่อ ragnarokmvptimer เข้ากับ Hermes Agent

โปรเจกต์นี้มีศักยภาพในการยกระดับจาก "เครื่องมือดูเวลา" (Static Timer) เป็น "ผู้ช่วยอัจฉริยะ" (Intelligent Assistant) โดยยังคงรักษาประสบการณ์การใช้งานเว็บแบบเดิมไว้ได้ 100%

---

## 1. กลยุทธ์การพัฒนา (Architecture Strategy)

อย่าพยายามให้ AI เข้าถึง React State โดยตรง แต่ให้ใช้ **API Layer** เป็นตัวกลาง:

*   **เว็บเดิม (UI):** ใช้งานได้ปกติผ่าน React Components
*   **Service Layer (Core Logic):** ย้าย Logic การรีเซ็ต/คำนวณ ไปไว้ที่ `src/services/`
*   **API/Tools Layer (Interface):** สร้างชั้น API (หรือ Function Calling) เพื่อให้ Agent เข้ามาสั่งงานได้

**ผลลัพธ์:** ทั้งผู้ใช้ (ผ่าน UI) และ Agent (ผ่าน API) จะเรียกใช้ `Service` เดียวกัน ทำให้มั่นใจได้ว่าข้อมูลตรงกันเสมอ

---

## 2. การใช้ AI ขนาดเล็ก (0.5B - รันบน Ollama)

โมเดลขนาดเล็กอย่าง 0.5B ไม่เหมาะกับการวางแผนซับซ้อน แต่เหมาะมากกับการเป็น **"Command Parser"**

*   **หน้าที่ของ AI:** รับภาษาธรรมชาติ (เช่น "รีเซ็ต MVP ตัวนี้ให้หน่อย") -> แปลงเป็น JSON คำสั่ง (เช่น `{"action": "reset", "mvp": "..."}`)
*   **หัวใจสำคัญ:** 
    *   **Few-Shot Prompting:** ป้อนตัวอย่างคำสั่งให้โมเดลดูบ่อยๆ
    *   **Strict JSON Mode:** บังคับให้ AI ตอบเป็น JSON ที่โปรแกรมอ่านได้ทันที
    *   **Validation:** เขียนโค้ดตรวจสอบฝั่ง API ให้แน่นหนา เพื่อป้องกัน AI ส่งคำสั่งที่ไม่มีอยู่จริง

---

## 3. ประโยชน์ที่ได้รับ (Use Cases)

เมื่อ Hermes Agent เข้ามาจัดการข้อมูล:

1.  **Proactive Notification:** แจ้งเตือนบอสเกิดผ่านช่องทางอื่น (Discord/Telegram) ได้โดยไม่ต้องเฝ้าหน้าจอ
2.  **Multi-Server Intelligence:** ให้ AI ช่วยเปรียบเทียบตารางบอสหลายเซิร์ฟเวอร์ และวางแผน Route ฟาร์มที่ดีที่สุดให้
3.  **Automated Data Maintenance:** ใช้ AI ช่วยอัปเดตไฟล์ `src/data/*.json` เมื่อมีการปรับ Patch เกม
4.  **Natural Language Control:** สั่งงานผ่านเสียงหรือแชท ขณะที่คุณกำลังเล่นเกมอยู่

---

## 4. Roadmap การพัฒนา (Step-by-Step)

| ลำดับ | สิ่งที่ต้องทำ | วัตถุประสงค์ |
| :--- | :--- | :--- |
| **Phase 1** | **Refactor Logic** | ย้าย Logic การรีเซ็ต/คำนวณ ไปที่ `src/services/` |
| **Phase 2** | **Build API/Tools** | สร้าง Interface ที่รับคำสั่ง JSON แล้วไปเรียก Service |
| **Phase 3** | **Agent Integration** | เชื่อม Ollama (0.5B) เข้ากับ API ที่ทำไว้ใน Phase 2 |

---

> **ข้อแนะนำสำคัญ:** การทำ API คือรากฐานที่สำคัญที่สุด ทำ API ให้เสร็จก่อน แล้วคุณจะสามารถเปลี่ยนโมเดล AI หรือเพิ่มความสามารถในอนาคตได้ง่ายโดยไม่กระทบเว็บหลักครับ
