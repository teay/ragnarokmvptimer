# Terminal-Based MVP Tracker Concept

แนวคิดการเปลี่ยนผ่านจาก WebApp ไปสู่ Terminal User Interface (TUI) เพื่อเน้นประสิทธิภาพสูงสุดและใช้ทรัพยากร Hardware ให้คุ้มค่าที่สุด

## 1. ทำไมต้องเป็น TUI (Terminal User Interface)?
การเปลี่ยนไปใช้แอปแบบ Text-based หรือ TUI มีข้อดีมหาศาลสำหรับสาย Dev ที่ต้องการความเร็ว:

*   **ประหยัดทรัพยากร:** ไม่ต้องเรนเดอร์ผ่าน Browser (Chromium) ทำให้ใช้ RAM และ CPU ต่ำมาก ระดับ native application ของแท้
*   **ความเร็วระดับ Real-time:** ข้อมูลที่สกัดได้จาก OCR สามารถอัปเดตลงหน้าจอ Terminal ได้โดยตรงโดยไม่มี Latency จากการเรนเดอร์ UI ซับซ้อน
*   **ความเรียบง่าย:** ตัดความซับซ้อนเรื่อง CSS/React ออกไป โฟกัสเฉพาะ Logic การจัดการข้อมูลและการแสดงผลที่สำคัญ (ชื่อบอสและเวลา) เท่านั้น

## 2. เปรียบเทียบ: WebApp เดิม vs. TUI ใหม่

| ฟีเจอร์ | WebApp (เดิม) | TUI (ใหม่) |
| :--- | :--- | :--- |
| **การใช้ทรัพยากร** | ปานกลาง (Chromium) | **ต่ำมาก (Native Rust)** |
| **ความไวในการอัปเดต** | มี Latency (IPC) | **Real-time (Memory-mapped)** |
| **ความซับซ้อนของ UI** | สูง (CSS/React) | **ต่ำ (Text-based)** |
| **การดูแลรักษา** | ยาก (ต้องแก้ CSS/React) | **ง่ายมาก (แก้ Logic Rust)** |

## 3. สแต็กเทคโนโลยีที่แนะนำ (Rust Ecosystem)
*   **Rust:** ภาษาหลักสำหรับควบคุมทุกอย่าง (Logic + AI)
*   **`ratatui` (Crate):** สำหรับวาด UI บน Terminal ให้เป็นตารางที่สวยงามและอ่านง่าย
*   **`notify` (Crate):** สำหรับเฝ้าดู Folder Screenshot ตลอดเวลา
*   **`leptess` / `tesseract-rs`:** สำหรับทำ OCR ดึงข้อความจาก Screenshot

## 4. Roadmap สำหรับการพัฒนา
1.  **File Watcher:** เขียนโปรแกรม Rust ให้พ่นข้อความบน Terminal ทันทีที่มีไฟล์ใหม่ถูกบันทึก
2.  **OCR Integration:** เชื่อมต่อ OCR เพื่อสกัด "ชื่อบอส" และ "เวลา" ออกมาจากรูปภาพ
3.  **TUI Construction:** ใช้ `ratatui` สร้างตาราง MVP Timer บน Terminal
4.  **Data Logic:** ผูกข้อมูลที่ได้เข้ากับไฟล์ JSON ในเครื่อง เพื่ออัปเดตข้อมูลแบบอัตโนมัติ
