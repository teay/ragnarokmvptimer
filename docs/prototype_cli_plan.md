# Prototype Plan: CLI File Watcher & OCR

แนวทางการสร้างโปรโตไทป์ขนาดเล็ก (CLI) เพื่อทดสอบแนวคิด "Auto-Log จาก Screenshot" ก่อนที่จะตัดสินใจพัฒนาเป็น GUI เต็มตัว

## 1. วัตถุประสงค์ของโปรโตไทป์
*   **เพื่อทดสอบ:** การทำงานของ `notify` (File Watcher) ในการตรวจจับไฟล์ใหม่ในโฟลเดอร์ Screenshot ของเกม Ragnarok
*   **เพื่อทดสอบ:** ความแม่นยำของการอ่านข้อความ (OCR) จากภาพ Screenshot โดยใช้ไลบรารี Rust
*   **เพื่อประเมิน:** ความคุ้มค่าในการลงทุนเวลาพัฒนาต่อเป็นแอปพลิเคชัน GUI เต็มรูปแบบ

## 2. ขั้นตอนการทำงานของโปรโตไทป์
1.  **Watcher Module:** เริ่มต้นโปรแกรมแล้วค้างไว้ (Idle) เพื่อรอเหตุการณ์ `Create` ในโฟลเดอร์ `ScreenShot` ของ Ragnarok
2.  **Trigger:** ทันทีที่มีไฟล์ใหม่ (.png/.jpg) ปรากฏขึ้น ให้โปรแกรมแสดงข้อความใน Terminal ว่า "Detected new file: [filename]"
3.  **OCR Module:** ส่งภาพที่ตรวจพบเข้าสู่ระบบ OCR เพื่ออ่านข้อความ
4.  **Logging:** พิมพ์ผลลัพธ์ที่อ่านได้ (เช่น ชื่อบอส, เวลา) ออกมาที่หน้าจอ Terminal เพื่อตรวจสอบความแม่นยำ

## 3. สิ่งที่ต้องเตรียมในโปรเจกต์ (Rust Dependencies)
เราจะเพิ่ม crate เหล่านี้เข้าไปใน `Cargo.toml`:
*   `notify`: สำหรับเฝ้าดูระบบไฟล์
*   `tesseract-rs` หรือ `leptess`: สำหรับทำ OCR (ต้องติดตั้ง Tesseract Engine ในเครื่องก่อน)
*   `tokio`: สำหรับจัดการงานแบบ Asynchronous

## 4. ประโยชน์ของแนวทางนี้
*   **ลดความเสี่ยง:** ไม่เสียเวลาทำ UI/UX ถ้าตัว Core Logic (OCR) ยังไม่แม่นยำพอ
*   **Speed:** โค้ดส่วนนี้มีขนาดเล็ก พัฒนาเสร็จในไม่กี่ชั่วโมง
*   **Validation:** เป็น "ตัวตัดสิน" ว่าไอเดียนี้จะไปต่อหรือพอแค่นี้
