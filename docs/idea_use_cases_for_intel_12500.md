# Idea Use Cases for Intel 12500

## 1. "AI-Powered MVP Detection" (ใช้ iGPU + OpenVINO)
แทนที่จะให้ผู้ใช้มานั่งกด "บันทึกเวลา" เอง (Manual Click) คุณสามารถสร้างฟีเจอร์ Auto-Capture MVP Kill ได้ครับ
* **วิธีการ:** พัฒนาตัวจับภาพหน้าจอ (Screen Capture) ที่รัน Background แล้วส่งเฟรมภาพเข้าไปให้โมเดล Object Detection เล็กๆ รันบน **iGPU** ของ i5-12500
* **ตัวตรวจจับ:** ฝึกโมเดลให้จำ "เอฟเฟกต์ตอนบอสตาย" หรือ "ไอเทมดรอป" หรือ "หลุมศพบอส" (จาก `mvp_tomb.png`)
* **ความเหนือกว่า:** ผู้ใช้แค่เปิดจอทิ้งไว้ เมื่อบอสตายในจอ ระบบจะตรวจจับภาพและ "อัปเดตเวลาให้โดยอัตโนมัติ" โดยแทบไม่กิน CPU หลักของเครื่อง

## 2. "Voice Command & Notification" (ใช้ GNA 3.0)
เปลี่ยนจากการแจ้งเตือนผ่านเสียงแจ้งเตือนแบบเดิมๆ เป็นการสั่งการด้วยเสียง
* **วิธีการ:** ใช้ **GNA 3.0** รันโมเดล Speech Recognition ขนาดเล็ก (เช่น Whisper-tiny หรือ Vosk) เพื่อคอยฟังคำสั่งเสียง
* **ฟีเจอร์:**
    * สั่งด้วยเสียง: "MVP 1 เหลืออีกกี่นาที?" หรือ "MVP ลอร์ดไนท์ตายแล้ว"
    * ระบบจะตอบกลับ (Text-to-Speech) หรือส่งเสียงเตือนผ่านระบบ GNA ซึ่งจะเงียบเชียบและประหยัดพลังงานมากที่สุด
* **ความเหนือกว่า:** คุณไม่ต้องพับหน้าจอเกมมาดูเวลา สามารถใช้เสียงสั่งในขณะที่กำลังตีมอนสเตอร์ได้

## 3. "Predictive Respawn with Trend Analysis" (ใช้ AVX-VNNI)
ฟีเจอร์นี้เหมาะสำหรับสาย Hardcore ที่ต้องการแม่นยำสูง
* **วิธีการ:** บอสตายเวลาเดิมๆ บ่อยครั้ง (เช่น วงรอบบอสในเซิร์ฟ) คุณสามารถเขียนโมเดลสถิติง่ายๆ (Linear Regression หรือ Time-series analysis) เพื่อทำ **Respawn Prediction**
* **การใช้ AVX-VNNI:** ถ้าคุณมีฐานข้อมูลการตายของบอสย้อนหลังจำนวนมาก (ใน `src/data/*.json`) คุณสามารถใช้ชุดคำสั่ง **AVX-VNNI** คำนวณสถิติความน่าจะเป็นในการเกิดของบอสในแต่ละวัน/เวลา เพื่อแจ้งเตือน "ช่วงเวลาที่บอสมีโอกาสเกิดสูง" (Hot Zone) ได้อย่างรวดเร็วมาก
* **ความเหนือกว่า:** แทนที่จะแค่บอกเวลาตายตรงๆ แอปจะ "พยากรณ์" ให้ด้วยว่าบอสตัวไหนน่าจะเกิดใน 5 นาทีข้างหน้า

## แนวทางการพัฒนา (สำหรับ Terminal-based / Tauri)
เนื่องจากคุณใช้ Tauri (Rust) คุณอยู่ในจุดที่ได้เปรียบมากครับ:

1. **Rust + OpenVINO:** มี [openvino-rs](https://github.com/intel/openvino-rs) ซึ่งเป็น bindings สำหรับ Rust คุณสามารถเขียน Rust โค้ดเรียกใช้โมเดล AI บน iGPU ของ 12500 ได้โดยตรง
2. **Terminal Base (TUI):** คุณสามารถใช้ **Ratatui** (ไลบรารี Rust สำหรับสร้าง Terminal UI) มาสร้างหน้าจอ MVP Timer ที่สวยงามและรวดเร็วมากใน Terminal
3. **Cross-Platform Architecture:** ฟีเจอร์ AI จะรันเป็น "Sidecar Process" ใน Rust ที่สื่อสารกับ Frontend ผ่าน Tauri IPC

## อธิบายการทำงานของ AI-Powered MVP Detection
การทำงานของ **AI-Powered MVP Detection** ที่รันบน i5-12500 ผ่าน Tauri (Rust) จะไม่ได้ทำแบบ "ถึกๆ" แต่เป็นกระบวนการแบบ Pipeline ที่เน้นประสิทธิภาพครับ โดยแบ่งการทำงานออกเป็น 4 ส่วนหลักดังนี้ครับ:

### 1. Screen Capture Pipeline (The "Hook")
แทนที่จะจับภาพหน้าจอทั้งหมด (ซึ่งกิน CPU มหาศาล) เราจะใช้ Native API ของ OS (เช่น `dxgi-capture` บน Windows) ผ่าน Rust เพื่อ "Crop" เอาเฉพาะพื้นที่ที่เราสนใจ (Region of Interest) เช่น บริเวณที่มักจะเห็นหลุมศพบอส หรือช่องแชท/Log
* **Trick:** เราจะจับภาพแค่ 2-5 เฟรมต่อวินาที (FPS) ก็พอ เพราะบอสตายไม่ได้เกิดขึ้นในเสี้ยววินาทีจนจับไม่ทัน การลด FPS ช่วยลดภาระการประมวลผลไปได้มหาศาลครับ

### 2. The AI Inference Pipeline (The "Brain")
นี่คือส่วนที่ดึงพลัง **iGPU** ออกมาใช้ครับ โดยเราจะใช้ **OpenVINO™** เข้ามาจัดการ
* **Model Optimization:** เราจะนำโมเดลที่เราเทรนมา (เช่น YOLOv8-tiny หรือโมเดลจำแนกภาพแบบ Custom) มาแปลงเป็น **OpenVINO Intermediate Representation (IR)**
* **Hardware Selection:** ในโค้ด Rust ของเรา เราจะสั่ง `ov_core_select_device("GPU")`
* **Execution:** เมื่อภาพถูกส่งเข้า iGPU, ข้อมูลจะถูกประมวลผลภายใน **Execution Units (EUs)** ของ UHD 770 โดยตรง ข้อมูลภาพจะถูกแปลงเป็น Tensor และวิ่งผ่าน Neural Network ในหน่วยความจำของ GPU ทำให้ CPU แทบไม่ต้องทำอะไรเลยนอกจาก "รอรับผลลัพธ์"

### 3. Logic & Decision Making (The "Dispatcher")
เมื่อ iGPU ประมวลผลเสร็จ มันจะส่ง "Confidence Score" (คะแนนความมั่นใจ) กลับมาหา Rust:
* ถ้าผลลัพธ์บอกว่า: "ตรวจพบหลุมศพ (Tomb) ด้วยความมั่นใจ 95%"
* โปรแกรม Rust จะตรวจสอบว่า MVP ตัวนั้นถูก "Kill" ไปแล้วหรือยัง หรือต้องอัปเดตเวลาหรือไม่
* จากนั้นค่อยยิง Event ไปที่ UI (Frontend) ให้ทำการ Update Timer หรือแจ้งเตือน

### 4. ทำไม i5-12500 ถึงทำได้แบบ "ชิลๆ"?
* **Async Processing:** เราใช้ Rust `tokio` (Async runtime) จัดการให้การถ่ายภาพหน้าจอและการตรวจจับภาพ แยกส่วนออกจาก Thread ที่แสดงผลหน้าจอ UI (Main Thread) ทำให้เกม RO ของคุณไม่กระตุกเลย
* **Zero-Copy Memory:** ถ้าใช้ OpenVINO ร่วมกับ Intel iGPU ข้อมูลภาพที่ถ่ายมาสามารถส่งต่อไปยัง GPU ได้ด้วยเทคนิคที่แทบไม่ต้องก๊อปปี้ข้อมูลซ้ำซ้อนใน RAM (Shared Memory)
* **Quick Sync Hardware:** หากอนาคตคุณพัฒนาให้แอป "อัดวิดีโอ" จังหวะบอสตายเก็บไว้ด้วย ตัวเครื่องยังมี **Intel Quick Sync** ช่วย Encode วิดีโอได้โดยไม่ดึง CPU เลยแม้แต่น้อย

### ภาพรวม Workflow ของโปรแกรม:
1. **[Rust/Tauri]** -> `[Screen Capture Module]` (ดึงภาพเฉพาะส่วน)
2. **[Rust -> OpenVINO Plugin]** -> ส่ง Tensor ให้ `[iGPU (UHD 770)]`
3. **[iGPU]** -> คำนวณ (Inference) -> ส่งค่ากลับมาว่า "เจออะไร"
4. **[Rust]** -> ตรวจสอบเงื่อนไข -> `[Update Application State]` -> `[UI Update]`
