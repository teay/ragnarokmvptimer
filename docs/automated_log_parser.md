# Automated Log Parser for Ragnarok

แนวทางการพัฒนาระบบอัตโนมัติ โดยการเฝ้าดูโฟลเดอร์ Screenshot ของเกม เพื่อดึงข้อมูลบอสที่ตายและเวลาที่ตายโดยอัตโนมัติ

## 1. วิธีการทำงาน (The "Observer" Pattern)
เราจะใช้วิธีการเฝ้าดูการเปลี่ยนแปลงของระบบไฟล์แทนการประมวลผลวิดีโอแบบ Real-time:

*   **File System Watcher (Rust/Tauri):** ใช้ crate `notify` เพื่อเฝ้าดูโฟลเดอร์ `ScreenShot` ของเกม RO ทันทีที่มีไฟล์ใหม่ถูกสร้างขึ้น ระบบจะรับสัญญาณอัตโนมัติ
*   **OCR (Optical Character Recognition):** แทนที่จะพึ่งพา AI ตรวจจับวัตถุ (Object Detection) เราจะใช้ OCR (เช่น EasyOCR หรือ Tesseract) เพื่อ "อ่าน" ข้อความจากภาพ Screenshot ที่เกมบันทึกไว้
    *   ระบบจะทำการหา Keyword เช่น `"ถูกกำจัดโดย"`, `"[ชื่อบอส]"`, หรือ `"เวลา:"`
*   **Database Update:** เมื่ออ่านค่าได้แล้ว โปรแกรมจะใช้ Regex (Regular Expression) สกัดข้อมูล "ชื่อบอส" และ "เวลาที่ตาย" แล้วนำไปอัปเดตไฟล์ข้อมูล JSON ของโปรเจกต์โดยตรง

## 2. จุดเด่นของแนวทางนี้
*   **ประหยัดพลังงาน:** โปรแกรมจะอยู่ในสถานะ "รอ (Idle)" ตลอดเวลา ไม่กิน CPU จนกว่าจะมีไฟล์ใหม่เกิดขึ้น
*   **ความแม่นยำสูง:** การอ่าน Text จากภาพนิ่งมีความแม่นยำสูงกว่าการประมวลผลจากวิดีโอเคลื่อนไหว
*   **ปลอดภัย:** ไม่ต้องเข้าไปยุ่งกับ Memory ของเกม ทำให้ลดความเสี่ยงต่อระบบ Anti-cheat

## 3. การนำความสามารถของ Intel 12500 มาใช้
*   **iGPU (UHD 770):** ใช้ OpenVINO เร่งความเร็วในการประมวลผล Image Pre-processing และ OCR ทำให้การอ่านไฟล์ทำได้รวดเร็วทันใจ
*   **AVX-VNNI (DL Boost):** หากต้องการประมวลผลภาพย้อนหลังจำนวนมาก (เช่น อ่าน Screenshot เก่าๆ 100 ภาพ) ชุดคำสั่งนี้จะช่วยให้การปรับแต่งภาพ (Grayscale, Contrast) ทำได้เร็วขึ้นหลายเท่า
*   **Async Processing (Rust):** การใช้ `tokio` (Async runtime) ของ Rust ช่วยให้การเฝ้าดูไฟล์ทำงานแยกจาก UI หลัก ทำให้ไม่มีผลกระทบต่อประสิทธิภาพการเล่นเกม
