# Developer Performance & Readiness Assessment
**Date:** March 10, 2026
**Project:** Ragnarok MVP Timer (Tauri + React + TypeScript)

## 1. การแก้ปัญหาเชิงเทคนิค (Technical Problem Solving)
- **ทักษะ:** ความสามารถในการวิเคราะห์ปัญหาที่เกิดจากโครงสร้างระบบ (Platform-Specific Issues)
- **หลักฐาน:** วิเคราะห์พบข้อจำกัดของ Browser Engine (WebkitGTK) ในแอป Desktop (Tauri) ที่ไม่รองรับการกด `Tab` ในช่อง `datetime-local` และตัดสินใจเปลี่ยนแนวทางมาสร้าง **Custom Segmented Input** เพื่อแก้ปัญหาให้ขาด 100%
- **ระดับความสามารถ:** **Senior Level** (ไม่ได้มองแค่ตัวโค้ด แต่มองไปถึงระดับ Engine และหาวิธี Workaround ที่ยั่งยืน)

## 2. การควบคุมคุณภาพและ UX (Quality Assurance & UX Attention)
- **ทักษะ:** สายตาที่เฉียบคมและการใส่ใจในรายละเอียด (Eye for Detail)
- **หลักฐาน:**
    - สังเกตเห็นปัญหาการคำนวณผิดพลาดเมื่อระบุปีที่ไกลเกินไป (Edge Case: Year 9999)
    - พบความไม่ชัดเจนของ UI (Focus Highlight ฟุ้งกระจายเกินไป) และสั่งให้ปรับจูนจนดูสะอาดตาและใช้งานง่าย (Sharp Outline)
    - ตรวจสอบระบบการกรอกตัวเลข (Typing Logic) จนพบว่าระบบ Auto-jump เดิมขัดขวางการพิมพ์เลข 2 หลัก
- **ระดับความสามารถ:** **Very Strong** (มี Product Mindset สูง ไม่ยอมปล่อยให้งานที่มีจุดบกพร่องเล็กน้อยผ่านไป)

## 3. กระบวนการทำงานแบบมืออาชีพ (Professional Workflow)
- **ทักษะ:** การจัดการ Software Development Life Cycle (SDLC)
- **หลักฐาน:**
    - ใช้ระบบ **Git Branching** ในการแยกฟีเจอร์ใหม่เพื่อทดสอบ
    - ใช้ **GitHub Actions (CI/CD)** ในการ Build ไฟล์จริง (.exe) มาทดสอบก่อนการ Merge เข้า Main
    - การ Merge และ Deploy ขึ้นหน้าเว็บจริง (Production) อย่างเป็นระบบ
- **ระดับความสามารถ:** **Professional Standard** (ทำงานเหมือนนักพัฒนาในบริษัทเทคโนโลยีชั้นนำ)

## 4. ความคิดเชิงผลิตภัณฑ์ (Product & Business Logic)
- **ทักษะ:** การคิดวิเคราะห์บนพื้นฐานความเป็นจริง (Real-world Pragmatism)
- **หลักฐาน:** ตัดสินใจจำกัดช่วงปี (2020-2035) แทนการเผื่อไว้ 100 ปี เพราะวิเคราะห์แล้วว่า "ไม่สมเหตุสมผล" ต่ออายุขัยของเกมและการใช้งานจริง
- **ระดับความสามารถ:** **Strong** (ไม่ได้ทำงานตามสั่งเพียงอย่างเดียว แต่ช่วยขัดเกลา Logic ให้สมเหตุสมผล)

## 5. ความทรหดในการแก้บั๊ก (Debugging Resilience)
- **ทักษะ:** การรับมือกับข้อผิดพลาดของระบบ Build และ Environment
- **หลักฐาน:** ร่วมกันแก้ปัญหา Linaria Build Error ที่เกิดขึ้นซ้ำซ้อนในระบบ GitHub Actions จนสามารถทำให้โปรเจกต์ Build ผ่านได้ทุก Platform
- **ระดับความสามารถ:** **Ready for Work** (มีความอดทนและใจเย็นในการไล่แก้ปัญหาทีละจุดจนจบงาน)
